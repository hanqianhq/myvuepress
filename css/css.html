# 盒子模型

每一个 HTML 标签都是一个方块

然后这个方块又包着几个小方块，如同盒子一层层的包裹着
这就是所谓的盒模型

css 的盒模型由 content(内容)、padding(内边距)、border(边框)、margin(外边距)组成

盒子的大小为 content+padding+border 即内容的(width)+内边距的再加上边框，而不是加上 margin

但盒子的大小由 content+padding+border 这几部分决定，把 margin 算进去的那是盒子占据的位置，而不是盒子的大小！

我们在编写页面代码时应尽量使用标准的 W3C 模型（需在页面中声明 DOCTYPE 类型），这样可以避免多个浏览器对同一页面的不兼容。

因为若不声明 DOCTYPE 类型，IE 浏览器会将盒子模型解释为 IE 盒子模型，FireFox 等会将其解释为 W3C 盒子模型

# IE 盒模型 （怪异盒模型）

我们大多数时会使用到的

宽高就是盒子宽高，border 和 padding 都在它的内部 box-sizing:border-box

box-sizing 属性

在 ie8+浏览器中使用哪个盒模型可以由 box-sizing(CSS 新增的属性)控制，默认值为 content-box，即标准盒模型

注意：

如果在 ie6,7,8 中 DOCTYPE 缺失会触发 IE 模式

# W3C 标准盒模型

记住：

属性 width，height 只包含内容 content，不包含 border 和 padding
若在页面中声明了 DOCTYPE 类型，所有的浏览器都会把盒模型解释为 W3C 盒模型

# BFC

# 理解 BFC

Block fomatting context 直译过来就是，块级格式化上下文 什么意思？

我们都知道，页面是由 css 布局和若干个 BOX 组成的

元素类型和 display 属性，决定了 Box 的类型 而不同的 Box 又会参与不同的 fomatting context

Fomatting Context 是一个独立的渲染区域，有自己的一套渲染规则 规定了内部元素该如何定位，以及它们之间的相互作用 通俗来讲，BFC 就是一个容器，里面装的都是块级元素

# 如何创建 BFC

  • float 浮动元素
  • overflow：hidden|auto|scroll 不为 visible
  • display：table-cell|table-caption|inline-block|flex
  • position：absolute|fixed
  • 根元素

# 水平垂直居中

# 定位1：宽高具体时

// 盒子宽100px 高50px

.box{
    position:absolute;
    top:50%;
    left:50%;
    margin-top:-25px;
    margin-left:-50px;
}

为什么这里要加 margin 呢？因为绝对定位是根据盒子的左上角来定的，要做一点偏移

# 定位2：不考虑宽高时

如果盒子的宽高不具体时怎么办呢？

.box{
	position:absolute;
    top:0;
    left:0;
    right:0;
    bottom:0;
    margin:auto;
}

这种情况适用于盒子有宽高，但是我们可以不考虑宽高的情况

# 定位3：变形位移

.box{
	position:absolute;
    top:50%;
    left:50%;
    transform:translate(-50%，-50%)
}

需要移动，但不需要计算，因为用了CSS3，所以兼容性不是很好

# flex 布局实现

如果不想基于定位，可以用这种方式来实现

body{
    display:flex;
    justify-content:center;
    align-items:center;
}

同样要考虑兼容性问题

# 纯 JS 实现

let HTML = document.documentElement,
    winW = HTML.clientHeight,
    winH = HTML.clientHeight,
    boxW = box.offsetWidth,
    boxH = box.offsetHeight;

box.style.position = "absolute";
box.style.left = (winW-boxW)/2 + 'px';
box.style.top = (winH-boxH)/2 + 'px';

# table-cell

这个方案本身是用来控制文本的，但是子元素盒子不是文本
怎么办，我们就把盒子变成 inline

body{
    display:table-cell;
    vertical-align:middle;
    text-align:middle;
    // 宽高不能是百分比,必须是有固定宽高
    width:500px;
    height:500px;
}
.box{
	display:inline-block;
}

这种方法我们现在通常不太使用

# 响应尺寸

深刻理解

他们都是灵活可以扩展的单位
你使用 1rem 或者 1em ，其实都可以被浏览器翻译成 16px-160px 以及其他任何值

浏览器使用 px，那 1px 就显示为 1px

  • rem：如果默认根元素字体为 16px，那你的 10rem 就会渲染成 160px
  • em：如果一个 div 有 18px 字体大小，那此时你的元素 10em 就是 180px

浏览器默认虽然是 16px，但是浏览器的这个默认参数，用户是可以自己改变的 它支持 9-72px 之间的任何值！！

# rem

基于 html 元素字体大小

html 根元素 继承的就是 浏览器设置的字体大小！除非重写一个具体单位来覆盖 所以表面上 rem 根据的是 html 根元素的字体大小，但实际可能是来自浏览器的设置大小

# em

根据使用它的元素大小决定（其实不是根据父元素，是因为继承了父元素，所以造成误解）

# 选择器权重

# 权重的计算

  • !important
  • 行内样式
  • ID 选择器，权重：100
  • clas、属性选择器和伪类选择器，权重：10
    属性选择器指的是：根据元素的属性及属性值来选择元素，比如 button 的 type 属性等伪类选择器 :active :focus 等选项
  • 标签选择器和伪元素选择器，权重：1
    伪元素选择器 :before :after

# 权重之间的关系

!important > 行内样式 > ID 选择器 > 类选择器 | 属性选择器 | 伪类选择器 > 元素选择器

# 权重的注意

# 1、不推荐使用 !important

因为 !important 根本没有结构与上下文可言，并且很多时候权重的问题 就是因为不知道在哪里定义了一个!important而导致的

那么我们知道尽量不要使用 !important ,也应该知道如何覆盖它

虽然它的权重最高，但是 id 加上 !important 是可以覆盖 class 的 !important

# 2、行内样式总覆盖任何外部样式表样式，但会被!important 覆盖

# 3、联合组成，没有一个选择器属性高

这句要怎么理解呢？

无论多少个 class 组成的选择器，都没有一个 ID 选择器权重高 无论多少个 ID 组成的选择器，都没有行内样式权重高 无论多少个元素组成的选择器，都没有一个 class 选择器权重高

# 4、元素上的选择器不同的时候，计算权重高的则生效

.test #test{}   // class 10 + id 100 = 110
.test #test span{}   // class 10 + id 100 + span 1 = 111
.test #test .sonClass{}   // class 10 + id 100 + class 10 = 120

# 5、权重相同时，与元素距离近的选择器生效

#content h1 { // css样式表中
  padding: 5px;
}
<style type="text/css">
  #content h1 { // html头部 因为html头部离元素更近一点，所以生效
    padding: 10px;
  }
</style>

最后总结建议：

  • 尽量避免使用 !important
  • 利用 id 增加选择器权重
  • 减少选择器的个数（避免层层嵌套）

# 浮动布局

# 理解浮动

浮动会让元素脱离文档流，背景并没有被撑开，这就是我们说的文档塌陷
此外还影响父元素的同级元素，最终造成了页面文档流混乱

# 浮动布局

# 丢失空间

如果只给第一个元素设置浮动，第二个元素不设置，后面的元素会占用第一个元素空间。

# 并排展示

如果两个元素都设置浮动，它们会并排显示

# 浮动转块

元素浮动后会变为块元素包括行元素如 span，
所以浮动后的元素可以设置宽高

# 清除浮动

# 末尾添加空元素标签 clear:both

<span>浮动</span>
<span>浮动</span>
<span>浮动</span>
<span>浮动</span>
<span>浮动</span>
<span>浮动</span>
<span>浮动</span>
<span>浮动</span>
<span>浮动</span>
<span>浮动</span>
<span>浮动</span>
<p style="clear:both"></p>

# 父元素添加 overflow:hidden 或者 overflow:auto

div{
    overflow:hidden;
}

限定内容溢出元素框的情况，会计算出浮动流的高度

# 父元素添加浮动

div{
    float:left; // 或者 right
}

这种做法的缺点就是影响了正常文档流

# 使用伪元素（推荐）

<div class="clearfix">
...
</div>

<style>
.clearfix::after{
    content:'',
    display:block;
    clearfix:both;
}
</style>

相当于在容器末尾添加了空元素

# 定位布局

# 默认定位 static

position 的默认值，css 默认的布局方式，默认的文档流

# 相对定位 relative

位置偏移
元素所占空间还在原位
其他元素不会挤占他原有的空间

# 绝对定位 absolute

移除正常的文档流，其他元素会挤占它原来的空间

根据父级元素 Postion 进行偏移，
如果所有的父级都没有postion，那就按照 html 容器来进行偏移

常见于弹窗

# 固定定位 fixed

它的包含元素是浏览器窗口
它设置好偏移后，无论页面怎么滚动，它都会固定在同一位置

适用于固定浮窗，导航条

# 粘性定位 sticky

相当于 relative 和 fixed 结合体

它会保留原有的文档流，
让元素距离浏览器窗口定位时，固定在这个位置

nav{
  position:sticky;
  top:0;
}

  • IE 下，可以使用获取常规属性的方法来获取自定义属性，也可以使用 getAttribute() 获取自定义属性
  • Firefox 下，只能使用 getAttribute() 获取自定义属性

可通过加入 CSS 属性 -webkit-text-size-adjust: none; 解决
