# 基础类型

为什么需要数据类型？

计算机中，不同数据占据的存储空间是不同的
为了充分利用空间，需要定义不同数据类型

当我们声明一个变量时

var x = "变量"

实际是在内存空间中开辟了一块地方来存储这个数据
但是我们调用这个变量的时候不能靠输入十六位进制来取值吧，太麻烦了
所以起了一个变量名，实际就是在引用内存地址

# String

字符串类型是使用非常多的数据类型，也是相对简单的数据类型。

# Number

常规包含：

  • 浮点类、整数
  • 二进制
  • 十六进制

三个特殊值：

  • Infinity，无穷大，大于任何值
  • -Infinity，无穷小，小于任何值
  • NaN，非数值

# Boolean

  • true == 1
  • false == 0

注意

javascript 中任何值都可以转换为布尔值。

# Undefined

undefined 声明了变量，但没有赋值，它存在，但没有值
调用函数时，应该提供的参数没有提供，该参数就是 undefined

var a ;  // undefined

# Null

没有值，根本就不存在，没有任何可读的东西
作为函数的参数，表示该函数的参数不是对象

# instanceof

instanceof 运算符用于检测构造函数的 prototype 属性是否出现在某个实例对象的原型链上。

也可以理解为是否为某个对象的实例，typeof不能区分数组，但instanceof则可以。

let hd = [];
let houdunren = {};
console.log(hd instanceof Array); //true
console.log(houdunren instanceof Array); //false

let c = [1, 2, 3];
console.log(c instanceof Array); //true

let d = { name: "houdunren.com" };
console.log(d instanceof Object); //true

function User() {}
let hd = new User();
console.log(hd instanceof User); //true

# 作用域

:::作用域 变量与函数的可访问范围 :::

# 什么是作用域

function Fn(){
    var inX = "内部变量";
}
Fn(); // 这个函数是可以执行的
console.log(inX); // Uncaught ReferenceError: inX is not defined

作用域就是这样一个独立的地盘，变量是不会泄露出去
所以不同作用域下的变量名是不会冲突的

var n = 2;
function fn(){
  a = 1;
  return a;
}

console.log(fn());  // 1
console.log(a);  // 1

上面这个例子中，如果一个变量没有声明直接就赋值了，那么它就是一个全局变量

# 块作用域

ES5 只有函数作用域和全局作用域

所以经常发生变量提升，引起了作用域污染
（内部变量覆盖外部变量）


### 作用域污染
for(var i = 0; i < 10; i ++){
    console.log(i);
}
console.log(i); // 10

这就是作用域污染

我们看到很多jquery代码，写的都是：

(function(){...})()

这是一个自执行函数
目的就是为了防止内部变量泄露造成对外污染

# ES6 的块作用域

ES5 不存在块级作用域

大括号{}、小括号、if、for 这些括号括起来的
都算是块级作用域

let 和 const 定义的变量，只能在块级作用域访问

不能跨块，更不能跨函数

{
    var a = 1;
    let b = 2;
    const c = 3;
    console.log(a); // 1
    console.log(b); // 2
    console.log(c); // 3
}
console.log(a); // 1
console.log(b); // b is not defined
console.log(c); // c is not defined

# 立即执行函数

有什么用？

封装函数，向外暴露接口，可以防止函数重名引起的问题

(function(){
  function func(){
    ...
  }
})()

func() // error 无法生效

自执行函数很好的帮我们解决了，调用别人类库，又怕和自己定义的函数重名这个问题

接下来我们尝试封装一个方法，这也是第三方类库常用的解决办法

(function(show){
  function hd(){
    ...
  }
  function show(){
    ...
  }
  window.js2 = { hd,show }
})

js.hd()

这样，我们就调用了第三方类库的方法

# 闭包

# 什么是闭包

理解闭包

函数里边再定义函数，内部函数可以访问外部函数作用域的变量。
这样如果外部函数不暴露内部函数，那么外部就不知道内部这个函数的存在，从而保护了内部函数的数据。

# 闭包的应用

闭包会形成了一个私有的函数，jQuery 就是一个例子

// 计算两个数的平方和
function squareSum(x,y){
	function square(x){
    return x*x;
  }
  return square(a) + square(b);
}

console.log(squareSum(2,3)) // 13

内部函数也可以作为一个返回值返回
这样外部的函数就形成了一个高阶函数（返回一个函数的函数）

function person(){
  let name = '萨尔';
  function getName(){
    return name;
  }
  return getName;
}
var getName = person();
console.log(getName); // 得到内部函数 getNum()方法
console.log(getName()); // 萨尔

上面这样相当于给 name 设置了只读属性
外界只能访问它的值，但不能修改，这起到了保护作用
